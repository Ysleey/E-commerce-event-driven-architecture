# KAN-4 - Inicializacion de Repositorios en GitHub

## Objetivo

Crear y preparar el repositorio remoto del proyecto para trabajar con una rama principal protegida y una rama de integracion.

## Alcance implementado

1. Se crea el repositorio remoto en GitHub.
2. Se clona el proyecto localmente para comenzar el desarrollo.
3. Se estructuran las ramas principales `main` y `develop`.
4. Se configura el archivo `.gitignore` para Java/Maven y Angular.

## Evidencia tecnica

- Captura de Jira con la subtarea KAN-4 en estado finalizada.
- Descripcion visible en Jira con el criterio de rama `develop` protegida como Definition of Done.

## Definition of Done

- El repositorio queda accesible desde GitHub.
- La rama `develop` existe y queda protegida para el flujo de trabajo.
- El proyecto queda listo para recibir la configuracion local y la base del backend.
