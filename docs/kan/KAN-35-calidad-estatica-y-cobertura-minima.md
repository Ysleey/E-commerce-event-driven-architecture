# KAN-35 - Calidad estatica y cobertura minima

## Objetivo

Introducir analisis estatico y umbrales minimos de cobertura para backend y frontend con reglas graduales orientadas a mantenibilidad.

## Alcance implementado

1. Se ejecuta analisis estatica en CI.
2. Se definen umbrales minimos de cobertura.
3. Se publican reportes visibles y trazables por build.
4. Se establece un proceso para gestionar deuda tecnica.

## Evidencia tecnica

- Captura de Jira con la subtarea KAN-35 en estado de trabajo.
- DoD visible en Jira: reglas activas en CI y reporte de cobertura disponible.

## Definition of Done

- Las reglas de calidad estatica quedan activas en la integracion continua.
- La cobertura minima queda documentada y verificada.
- La deuda tecnica inicial queda priorizada y gestionable.
